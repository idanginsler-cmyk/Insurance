היי [שם],

בהמשך לשיחה שלנו, אני מבקש לקבל גישה לכלי Claude Code לצורך פיתוח 3 פרויקטים שיכולים לייצר ערך משמעותי לארגון. בניתי דמו עובד לכל אחד מהם.

פרויקט 1: דשבורד זיהוי הונאות בתביעות

מה עושה: מערכת שמקבלת קובץ CSV של תביעות ומייצרת דשבורד אינטראקטיבי — 12 ניתוחים שונים (תדירות, ריכוזי ספקים, קבלות חשודות, רשתות קשרים), drill-down למספרי תביעות ספציפיים לתחקור במערכות התפעוליות, ומערכת תיעוד ומעקב לממצאים.

הבעיה שפותר: היום בדיקת תביעות חשודות נעשית ידנית, איטית, וללא מערכת מרכזית שמשלבת ניתוח + תיעוד.

כלים נדרשים: Claude Code (פיתוח מקומי בלבד) + Java 22 (כבר מותקן). ללא חיבור לאינטרנט

פרויקט 2: ניתוח מסמכים תפעוליים וזיהוי זיופים

מה עושה: כלי שמנתח מסמכים (קבלות, אישורים רפואיים, חשבוניות) — חילוץ מידע אוטומטי, זיהוי אנומליות (תאריכי שבת, סכומים חריגים, מספור רציף), וזיהוי סימני זיוף (פונטים לא אחידים, עריכות מטא-דאטה).

הבעיה שפותר: מסמכים נבדקים ידנית, זיופים מתוחכמים קשה לזהות בעין, ואין כלי שמרכז את כל סימני האזהרה במקום אחד.

כלים נדרשים: Claude Code + גישה ל-Claude API עם יכולת Vision. דורש אישור API
💡 במסגרת Claude Enterprise, גישה ל-API עשויה להיות כלולה בתוכנית — כדאי לבדוק מה מאפשר הפלאן.

פרויקט 3: ספר חיתום אינטראקטיבי

מה עושה: מחליף את ספר החיתום (מסמך Word של מאות עמודים) בכלי אינטראקטיבי עם שני מצבים:

1. שאלון מובנה — בחירת מחלה → שאלות → החלטת חיתום לכל כיסוי (30+ כיסויים)

2. מצב חכם (NLP) — המערכת קוראת את קובץ הוורד של ספר החיתום, וכשמזינים שאלה במלל חופשי — היא מחפשת בתוך המסמך את עץ ההחלטות הרלוונטי ומחזירה את התשובה מתוך הספר בלבד, ללא "המצאת" תשובות.

הבעיה שפותר: חיפוש ידני במסמך ארוך, זמן הכשרה ארוך לחיתומאים חדשים, וסיכון לטעויות.

כלים נדרשים: Claude Code + Java 22 למצב האינטראקטיבי עובד כבר היום. למצב NLP: דורש Claude API
